# Meshtastic Pager OS

![Концепт Pager OS](assets/pager-mode-banner.svg)

`Meshtastic Pager OS` — это отдельное направление прошивки на базе Meshtastic. Цель уже не в том, чтобы сохранить старый UI устройства. Цель в том, чтобы превратить `Heltec V4` в маленький и понятный приёмник сообщений, который работает как пейджер.

## Статус проекта

- Основа: форк прошивки Meshtastic
- Основная цель сборки: `heltec-v4`
- Путь настройки: официальное мобильное приложение по BLE
- UI на самом устройстве: сведён к поведению пейджера

Проект поддерживается как community work, без коммерческой поддержки и без финансовой выгоды. Значительная часть реализации и документации была сделана с помощью AI, потому что сопровождающий проекта работает вне embedded-разработки.

## Зачем это нужно

Не всем нужен карманный радиокомпьютер с кучей локальных экранов. Иногда нужен прибор, который:

- спокойно ждёт
- просыпается на входящий текст
- ясно показывает сообщение
- возвращается в экономичный режим

Типичные сценарии:

- монтажники и техперсонал
- высотные работы и риггинг
- страйкбол и тренировки
- туризм и outdoor-группы
- суровые условия, где простой пейджер удобнее перегруженного интерфейса

Идея простая: не усложнять то, что уже должно просто работать.

## Схема экрана

OLED разбит на три фиксированные зоны:

1. Шапка
Батарея и текущее время.

2. Центр
Крупный горизонтально прокручивающийся текст сообщения.

3. Нижняя зона
Используется только для служебных поверхностей, если они нужны логике пейджера.

Центральная область — приоритетная. Сообщение должно читаться быстро, даже если устройство висит на одежде, страховке или разгрузке.

## Логика сообщений

Текущее поведение пейджера строится вокруг строгой очереди:

- Входящее сообщение будит экран
- Текст крутится в центральной зоне
- Если во время активного показа приходит следующее сообщение, оно подхватывается следующим, а таймер активности обновляется
- Непрочитанные сообщения остаются в локальной очереди до подтверждения кнопкой
- Локальная история специально считается временной и может быть очищена полностью

Это устройство не должно быть архивом. Если пользователю нужна долгая история, она остаётся в мобильном приложении.

## Логика кнопки

- Короткое нажатие на спящем экране будит экран и открывает просмотр непрочитанных
- Следующие короткие нажатия подтверждают текущее сообщение и листают историю непрочитанных
- Долгое удержание очищает все локально сохранённые pager-сообщения

Кнопка здесь нужна как быстрый инструмент подтверждения, а не как средство лазить по меню.

## Логика LED

- Белый светодиод мигает, пока есть непрочитанные сообщения
- Сначала мигает быстрее
- Позже мигает реже, чтобы не тратить лишнюю батарею

Светодиод — это тихое напоминание, что сообщение ещё ждёт внимания.

## Что сохраняется совместимым

- BLE-подключение и настройка через стандартное приложение Meshtastic
- Каналы
- Direct Messages
- Существующий provisioning и flashing flow

## Что намеренно убрано из UX

- стандартная OLED-карусель экранов
- богатая локальная навигация по устройству
- настройки с устройства как основной сценарий

Этот форк намеренно узкий по смыслу. Он не пытается улучшить все сценарии upstream сразу.

## Прошивка из браузера

Предпочтительный сценарий для пользователя:

1. Открыть [Pager OS Web Flasher](https://chemobook.github.io/meshtastic-pager-mode/)
2. Использовать настольный Chrome или Edge
3. Выбрать `Heltec V4`
4. Прошить устройство прямо из браузера
5. Дождаться перезагрузки

Если устройство не определяется:

- сначала заменить USB-кабель
- закрыть программы, которые держат serial-порт
- попробовать другой USB-порт

## Локальная сборка

```bash
pio run -e heltec-v4
```

Упаковать сборку в release-подобную папку:

```bash
./bin/pager-package.sh heltec-v4
```

Готовые файлы складываются сюда:

- [../../release-work/firmware/heltec-v4](../../release-work/firmware/heltec-v4)

## Практические замечания

- Это не официальный релиз Meshtastic
- Проверка на реальном устройстве важнее любых теоретических предположений
- Активное направление проекта теперь называется `Pager OS`, даже если в старых путях репозитория ещё встречается `pager-mode`
