# Meshtastic Pager Mode Fork

![Pager Mode banner](assets/pager-mode-banner.svg)

> Неофициальный форк Meshtastic для тех, кому нужен более удобный режим чтения сообщений в стиле пейджера на маленьких экранах.

## О проекте

Это **форк** Meshtastic, а не официальный upstream-релиз.

Он сделан заинтересованными людьми без какой-либо финансовой выгоды и выложен просто потому, что может быть полезен другим:

- без бизнеса
- без инвесторов
- без монетизации
- просто как практичная community-работа

Значительная часть работы в этом форке была сделана с помощью **искусственного интеллекта**. Это сказано специально и честно. Автор проекта работает в другой области, и AI здесь был рабочим инструментом, который помог довести идею до реального результата.

## Что меняет этот форк

- Добавляет `Pager Mode` в классический small-screen UI
- Переносит ту же идею в `InkHUD` для поддерживаемых e-ink устройств
- Фиксирует устройство на выбранном DM или канале, пока активен pager mode
- Позволяет выйти из pager mode длинным нажатием
- Сохраняет состояние pager mode после перезагрузки
- Делает длинные сообщения более читаемыми
- Старается оставить изменения меньше, чем полный редизайн интерфейса

## Для каких устройств это сделано

Форк в первую очередь рассчитан на маленькие экраны:

- OLED-устройства Meshtastic
- e-ink устройства на `InkHUD`

Большие TFT-устройства здесь не были основной целью.

## Зачем всё это

Идея очень простая:

- сделать чтение сообщений удобнее
- не раздувать ветку без необходимости
- сохранить как можно больше поведения upstream
- поделиться результатом открыто

## Рекомендуемые цели сборки

Примеры:

```bash
pio run -e heltec-v3
pio run -e heltec-v4
pio run -e heltec-wireless-paper-inkhud
pio run -e t-echo-inkhud
```

Используйте тот `env`, который соответствует вашему устройству.

## Готовые прошивки в этом workspace

Локальная папка для собранных и разложенных артефактов:

- [../../release-work/README.md](../../release-work/README.md)

Чтобы упаковать актуальные сборки в эту папку:

```bash
./bin/pager-package.sh heltec-v3 heltec-v4
```

Скрипт копирует свежие артефакты из `.pio/build/<env>/` в удобные release-папки и при необходимости подтягивает OTA-образ для ESP32-S3.

## Как собрать свою прошивку

### Быстрый путь

```bash
pio run -e heltec-v3
pio run -e heltec-v4
```

### Упаковать результат

```bash
./bin/pager-package.sh heltec-v3 heltec-v4
```

### Прошить через форковый helper

```bash
./bin/pager-flash.sh --board heltec-v3 --port /dev/tty.usbmodemXXXX
./bin/pager-flash.sh --board heltec-v4 --port /dev/tty.usbmodemXXXX
```

Этот wrapper использует существующий `bin/device-install.sh`, но делает выбор готовых файлов для `Heltec V3/V4` заметно проще.

## Обычный путь для пользователя

Для большинства людей удобнее всего так:

1. Скачать готовую сборку из релиза или из подготовленной локальной папки.
2. Выбрать правильный набор файлов именно для своей платы.
3. Прошить обычным способом Meshtastic или через helper-скрипт этого форка.
4. Проверить pager mode на реальном устройстве.

## Путь для сопровождающего

1. Собрать нужные цели.
2. Проверить хотя бы одно реальное устройство.
3. Упаковать артефакты.
4. Публиковать только те платы, которые реально тестировались.

## Ограничения

- Это всё ещё community fork, поэтому проверка на реальном железе важна.
- Большие TFT-цели не были основным фокусом.
- Русские дефолты здесь практичны, но они заметно более специфичны, чем upstream.
- Главная цель этого форка — польза, а не абсолютное покрытие всех плат.
